# 最終レポート作成タスクリスト (3システム開発)

本リストは，レポート課題で要求されている「3つのWebアプリ開発」と「仕様書3部構成」を達成するための作業工程を整理したものである．

- [ ] I．システム構想・仕様書ドラフト（開発者向け）
  - [ ] 1．システムテーマの選定と仕様策定
    - [ ] Webアプリケーション3つのテーマを選定する（例：ラリーガ，〇〇，△△）．
    - [ ] 各システムのリソース（データ）構造を定義する．
    - [ ] 全システムで統一した操作（ページ遷移，HTTPメソッド）を検討し，定義する．
  - [ ] 2．開発者向け仕様書ドラフト作成 (3システム分)
    - [ ] システム1 (ラリーガ): spec．mdの内容を最終決定する．
    - [ ] システム2 (〇〇): データ構造，ページ遷移，機能詳細を記述する．
    - [ ] システム3 (△△): データ構造，ページ遷移，機能詳細を記述する．
    - [ ] 授業で説明していない技術の採用理由とその概要を記述する．

- [ ] II．プログラム開発と動作確認
  - [ ] 1．共通実装
    - [ ] 全てのJSファイルの先頭に `"use strict";` を記述する．
    - [ ] データは変数内に記録する（データベースは使用しない）．
  - [ ] 2．システム実装 (3システム分)
    - [ ] 各システム（チーム，選手，国など）のデータ一覧表示機能を実装する (`GET /リソース`)．
    - [ ] 各システムの詳細表示機能を実装する (`GET /リソース/:id`)．
    - [ ] 各システムのデータ**追加**機能を実装する (`POST /リソース`)．
    - [ ] 各システムのデータ**変更**機能を実装する (`PUT /リソース/:id`)．
    - [ ] 各システムのデータ**削除**機能を実装する (`DELETE /リソース/:id`)．
  - [ ] 3．動作確認とバグ修正
    - [ ] 全てのCRUD操作（一覧・詳細表示，追加，変更，削除）が仕様通りに動作するか確認する．
    - [ ] 追加・削除・編集後のリダイレクトとメッセージ表示を確認する．
    - [ ] 統一した操作（REST APIの原則）が守られているか確認する．

- [ ] III．最終仕様書作成（管理者・利用者向け）
  - [ ] 1．管理者向け仕様書 (1システム分)
    - [ ] サービスを立ち上げる人向けの内容を意識して構成を検討する．
    - [ ] システムのインストールから起動，終了までの手順を具体的に記述する．
  - [ ] 2．利用者向け仕様書 (代表1システム分)
    - [ ] IT知識のない一般人向けの内容を意識して構成を検討する．
    - [ ] 使用できる機能（一覧，検索，データ操作など），画面説明，操作手順を分かりやすく記述する．
  - [ ] 3．最終ドキュメント作成
    - [ ] 全ての仕様書（開発者向け3部，管理者向け1部，利用者向け1部）を**TeXで作成**する．

- [ ] IV．提出準備
  - [ ] 作成したソースコード一式を Git にコミットする．
  - [ ] GitHub リポジトリを作成し，ソースコードを登録する．
  - [ ] TeXで作成したレポート（仕様書）内に，**GitHubのURLを記載**する．
  - [ ] 最終レポート（仕様書ファイル）を提出する．
